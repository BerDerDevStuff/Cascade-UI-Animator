The One-Stop solution for UI Animations

Cascade UI Animator v2 utilizes a Dual-Paradigm Architecture. It offers a high-level "Fire-and-Forget" service for simplicity, and a low-level "Track-Based" system for object-oriented control.

Why use this?

Standard TweenService fights over properties. Cascade's v2 engine uses a Global Property Buffer. This means you can Play a "Hover", a "Click", and a "Damage Shake" all on the same button, at the same time, and they blend perfectly without flickering.

Example: A button handling a looping glow, a hover scale, and a click punch simultaneously.

-- Grab the service and start composing
local AnimationService = require(game.ReplicatedStorage.CascadeAnimations.AnimationService)

API Reference

Version 2 introduces the AnimTrack class and a unified Service API.

Service API (Standard)

.Play(Instance, Config, Options?) -> Signal

Starts an animation on a single instance. Returns a custom Signal that fires events.

Example: Connection to Signals

local signal = AnimationService.Play(button, Hover, { Reset = true })

signal:Connect(function(status)
    print("Animation Status:", status) -- "Completed", "Looped", or "Cancelled"
end)

.CreateTrack(Instance, Config, Options?) -> AnimTrack

New in v2: Returns an AnimTrack object for precise control. Does not auto-play.

local track = AnimationService.CreateTrack(button, ClickAnim)

track.Completed:Connect(function()
    print("Track finished!")
end)

track:Play()
task.wait(0.5)
track:Pause() -- Pause halfway

.Pause(Instance, Config) / .Resume(Instance, Config)

New in v2: Pauses or resumes an animation on an object without needing the track object.

.ClearInstance(Instance)

Crucial: Halts all active tracks on the instance and performs a Deep Reset. Every property touched by the animation (Size, Color, Rotation, etc.) is reset to its initial cached state.

.SetFrame(Instance, Config, FrameNumber)

New in v2: Snaps the object to a specific frame based on the Config's FPS settings.

.Suspend()

Disconnects the service from RunService, globally pausing the engine.

Track Object API

Returned by CreateTrack.

Methods

  • :Play() - Starts playback.
  • :Pause() - Freezes playback at current time.
  • :Stop() - Stops and triggers Cancelled signal.
  • :Scrub(time) - Jumps to specific time.
  • :SetSpeed(number) - Changes playback speed dynamically.
  • :Destroy() - Cleans up track (Service auto-removes it).

Events (Native Signals)

  • .Completed - Fires when animation finishes.
  • .Looped - Fires when a loop cycle restarts.
  • .Cancelled - Fires when manually stopped.
  • .MarkerReached - (Experimental) Reserved for future events.

Editor Features

A visual guide to the power tools inside Cascade UI Animator.

Axis Isolation & Stagger

Most editors lock the whole Size property. Cascade lets you animate Size.X independently of Size.Y. Combined with the Stagger runtime feature (shown here), you can create complex, fluid reveals.

Smart Inspector

Don't fight with input fields. The inspector accepts loose syntax like {0.5, 10} for UDim and automatically formats it to Roblox standards.

Multi-Select & Drag

Box-select multiple keyframes across different tracks and drag them as a group. Retiming complex animations takes seconds, not minutes.

Relative Mode (âˆ†)

The killer feature. Define animations as "offsets" (e.g., +10px size) rather than absolute values. This lets you layer hover effects on top of any layout without breaking it.

Non-Destructive Editing

Streamline your editing process with industry-standard Undo, Redo, Copy, Cut, and Paste. Duplicate keyframes across tracks or reverse mistakes instantly, ensuring a fluid and forgiving creative environment.

Customizable Keybinds

Tailor the editor to your muscle memory. Remap every shortcut in the Keybind Settings to match your preferred workflow, ensuring a seamless editing experience.

Save & Import System

Never lose progress. The robust IO System allows you to serialize animations to data strings, making it effortless to save backups or transfer animations between projects or team members.

Intuitive Keyframing

Capture motion naturally with Auto-Record, which logs changes made in the viewport directly to the timeline. For precise control, simply Double-Click on any track to insert a keyframe at the exact cursor position.
